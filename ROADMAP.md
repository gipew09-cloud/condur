# Condur — план развития и известные ограничения

Обновлено: 18.07.2026.

## Недавно сделано

- **Зажигание по напряжению бортсети.** «Двигатель работает» определяется по
  напряжению (генератор заряжает → ~28 В у 24-вольтовой сети; заглушен → ~25 В), а не по
  сырому биту `ign` — он у части трекеров залипает на «включено». Совпадает с тем, как
  зажигание показывает сам Stavtrack. См. `telemetry_service.engine_running_from_voltage`.
- **Фантомный простой на РЦ исправлен.** Время простоя считается от текущей непрерывной
  стоянки, а не от старого «приезда» (если прошлый «отъезд» потерялся на устаревшем GPS).
  См. `scheduler_jobs._downtime_started_at`.
- **Пробег из Wialon.** `totalDistance` (метры → км) пишется в `mileage_km`, чтобы сверка
  «одометр водителя vs GPS-пробег» работала и по wialon-машинам, а не только по EGTS.
- **Зажигание в смене.** Уведомление владельцу о старте/завершении смены содержит статус
  двигателя; в хронологии смены/рейса — отметки «завёл/заглушил двигатель». Появляются на
  новых сменах (нужны реальные переходы вкл↔выкл, которые даёт определение по напряжению).

## Ближайшее (код, по приоритету)

1. **Выезд без смены — надёжно.** Сейчас сигнал «машина едет без открытой смены»
   определяется проверкой раз в 10 минут (`vehicle_mixup_detector_job`) — короткие
   поездки между тиками проскакивают. Сделать событийным (реагировать в момент прихода
   точки со скоростью > 3 у машины без смены) с фильтром от скачков GPS. Владельцу —
   всегда; водителю — если машина за ним закреплена (`default_vehicle_id`).
   *Примечание:* уведомление водителю уже есть (`_remind_drivers_to_start_shift`), но
   только для закреплённого водителя.
2. **Защита порта приёма GPS.** Проверять пароль/ID при логине Wialon и рвать соединение
   с чужими — чтобы посторонний не мог слать фальшивые координаты/пробег.
3. **Гистерезис геозон РЦ — перепроверить.** Убедиться, что «приезд/отъезд» не срабатывают
   от дрожания GPS (частично уже есть; усилить по скорости/расстоянию при необходимости).

## Продукт (позже, по одному блоку)

4. **Кнопка «📍 Где машины» в боте владельца** — прислать позиции, статусы, время на
   точке и геозону прямо в мессенджер. Не блокируется, ценно, несложно.
5. **Контроль температуры груза.** Датчики в кузов сейчас не подключены (`temp1/temp2` =
   заглушка). Когда подключат: алерт не на «стало тепло» (открыли двери), а на **тренд**
   «растёт на ходу — возможна поломка холодильника».
6. **Мини-приложение Telegram** как формат богатого UI. Важно: контент мини-аппа
   грузится с нашего сервера и подчиняется тем же блокировкам, что и веб-кабинет (см.
   «Решение за владельцем»).

## Решение за владельцем (не код)

- **Блокировки РКН.** Веб-кабинет часто не открывается без VPN (зарубежный хостинг).
  Надёжно лечит только российский хостинг веб-части; Cloudflare — временная лотерея;
  боты Telegram работают всегда. Смена только домена и переезд в мини-апп сами по себе
  доступ не возвращают (хостинг остаётся заблокированным). Telegram-боты с российского
  хостинга работают (это миф, что нет).

## Сознательно НЕ делаем сейчас

- Свои GPS-трекеры «из Китая» (морока с установкой/SIM/поддержкой; готовое решение уже
  умеет буферизацию при глушении сети).
- Нативное iOS/Android приложение (избыточно для текущего масштаба).
- Микросервисы / очереди задач / Kafka (текущая схема тянет парк до десятков машин).
- ЭДО / 1С (пока автопарк мал — рано).
- Смена языка на C#/C++/Go (Python-стек полностью оправдан; узкое место — ввод-вывод, а
  не язык).

## Известные ограничения

- Данные о положении приходят от Stavtrack с задержкой ~1–3 минуты и иногда пачками
  (буфер трекера при потере сети). Наш приёмник обрабатывает пакет за доли секунды —
  задержка на стороне трекера/ретранслятора, не бота.
- Часть сигналов (выезд без смены и т.п.) определяется периодической проверкой, а не
  мгновенно (см. «Ближайшее» №1).
- Один процесс `condur` тянет боты + веб + планировщик. При росте нагрузки стоит развести
  веб и ботов на отдельные процессы (тяжёлый рендеринг страницы может кратко подвесить
  long polling ботов).

## Тесты и релиз

- Тестовое окружение: системный Python 3.9 не годится; venv через `uv --python 3.12`.
  Прогонять `pytest tests/` (сейчас 170 зелёных). На каждый баг — регресс-тест, проверять
  откатом фикса.
- Владелец грузит код на GitHub вручную (папки `app` и `tests`), Railway деплоит сам.
