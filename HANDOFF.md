## ✅ ЧТО УЖЕ СДЕЛАНО

### Бот водителя — упрощение через фича-флаги (всё в `app/config.py`)
- 10 флагов `FEATURE_*` (по умолчанию выключены, кроме `RECEIPT_OCR`, `ODOMETER_PHOTO`,
  `CARGO_GEOLOCATION`). Выключено = водитель не видит кнопку/шаг; включишь = появится.
  Документировано в `README.md`.
- Минимум кнопок: Начать/Конец смены, Новый рейс, Выехал, Сдал груз, SOS, Документ, Расход,
  «➕ Добавить смену/рейс» (оффлайн задним числом). «Статус» — только команда `/status`.
- Одна свободная машина → смена стартует сразу без выбора.

### Бот — ключевые потоки
- **Одометр:** водитель шлёт ТОЛЬКО фото (старт и конец); число км вписывает ВЛАДЕЛЕЦ по
  кнопке «📍 Указать пробег». Есть валидация (≤7 цифр, конец ≥ начала, ≤5000 км/смена) и
  одноразовость (повторно не вписать). Зарплата пересчитывается после ввода км.
- **Геопозиция** запрашивается на «Выехал» И «Сдал груз» (с кнопкой «Пропустить»).
- **Расход** доступен в любой момент, даже без открытой смены (`shift_id=None`).
- **Выручка рейса:** водитель указывает после сдачи груза («правило первого» — кто первый
  указал, тот зафиксировал; водитель НЕ перетирает уже указанное, владелец — главный,
  одобряет/меняет; все кнопки одноразовые). Двойного ввода больше нет.
- **Зарплата:** типы per_km / per_trip / percent / fixed_per_shift / **fixed_per_month** (оклад).
- Владелец: одобрить/изменить/отклонить расход; установка часового пояса (`/timezone`).

### Веб-кабинет — дизайн «Свежий путь»
- Базовый рескин на ВСЕХ страницах: шрифт Onest, палитра, белые карточки, таблицы-в-карточках,
  статус-плашки (фильтр `pillclass`/`statusru`), единые заголовки `.page-head`.
- **Дашборд** — собран 1:1 по макету (KPI, «Требуют внимания», график выручка/прибыль,
  пончик расходов, «машины в работе», превью карты).
- KPI-карточки сверху списков: Рейсы / Машины / Водители / Расходы / Финансы.
- Вход (`login.html`) оформлен карточкой с брендом.

### Веб-кабинет — функции
- Добавление и редактирование рейсов на сайте; правка выручки (прибыль пересчитывается).
- Прибыль рейса = выручка − топливо − прочие расходы (привязка расходов к рейсу).
- Правка расходов на сайте (сумма/категория/статус) + загрузка фото чека владельцем
  (хранится в Postgres, миграция `0004`).
- Документы по рейсу: просмотр ТТН + загрузка владельцем (Postgres, миграция `0003`).
- Примерное время в пути по маршруту (из истории).
- **Статистика по машине** (только владельцу): `GET /vehicles/{id}/stats` — пробег/рейсы/
  выручка/расходы/прибыль/маржа за месяц и за всё время + документы.
- Пагинация списка рейсов; русские статусы везде.

  один лист со всеми рейсами за период (1, 2, 3…), шапка с реквизитами **Исполнителя**

  - Старый `build_acts_workbook` (листы по РЦ) больше не используется — можно удалить при случае.
  - Доработки (2026-06-25, по фидбеку): (1) **переименован в «Акт сверки»** — параметр
    `title` у `build_act_101rs` (деф. «Акт»), на сайте поле «Название документа» (деф.

## ⛔️ ЧТО ОСТАЛОСЬ (приоритет сверху вниз)

### 1. ✅ Excel-акт 101 РС — СДЕЛАНО (настоящая форма)
Реквизиты Исполнителя/Заказчика, шапка, колонки формы, итог прописью, подписи — готово
(см. раздел Excel выше; страница `/requisites`, миграция `0006`). Остаточные нюансы:
- В описании строки РЦ = `Trip.destination` (что ввёл водитель/владелец). Чтобы стояли
  **полные адреса РЦ из образца** — нужен **Справочник РЦ** (задача №3): нормализовать
  destination → канонический адрес.
- В образце прицепа в описании НЕТ (только тягач + водитель), поэтому колонки «№ ТТН/прицеп»
  из старого плана не делали. Привязки «машина→водитель→полуприцеп» есть в `РЦ Адреса Спб.xlsx`
  (строки 20–22) — добавить позже, если заказчик попросит прицеп в акте.
- НО (УСН/ОСН) в образце не фигурирует — не вносили; добавить полем, если понадобится.
