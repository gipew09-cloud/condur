## Что уже работает (НЕ ломать)

- **GPS-приёмник** (`app/telemetry/egts_receiver.py`) понимает ДВА протокола на одном
  порту (hayabusa.proxy.rlwy.net:12826): EGTS (первый байт 0x01) и Wialon IPS (символ '#',
  парсер `app/telemetry/wialon.py`). Wialon даёт зажигание/датчики. Защиты: рассинхрон
  рвёт соединение; точка из прошлого не трогает VehicleState (`state_update_allowed`);
  лог не захлёбывается на дампах (подробно — только первые 3 пакета соединения).
- **Бот**: выручка «одна живая кнопка», анти-даблклик (PerUserLockMiddleware),
  кнопки меню не съедаются шагами FSM, меню команд только у бота владельца.
- **Веб**: хронология смены по GPS (стоял/ехал + имя РЦ), правка одометра в карточке
  смены, бейджи «В рейсе/В смене/свободна», аналитика складов на /stats
  («обычно приезд», «быстрее всего»), финансовые дни считаются по поясу владельца
  через `_owner_day`. ВАЖНО: выражение `_owner_day` строить ОДИН раз и
  переиспользовать в select/where/group_by — два вызова дают два bind-параметра и
  Postgres падает с GroupingError (тест: test_owner_day_group_by_uses_single_bind_param).
- Данные не удаляются: рейсы/деньги/события — вечно, GPS-точки — 180 дней.
- Удалённые водители — soft-delete, в фильтрах помечены «(удалён)».

## Правила работы (важно)

- Тестовое окружение: системный Python 3.9 не годится; пересоздавать venv через uv
  (`uv venv --python 3.12` + `uv pip install -r requirements.txt pytest
  pytest-asyncio aiosqlite`). Прогонять `pytest tests/` ТРИ раза — сейчас 149 зелёных.
- После правок: удалить __pycache__/.pytest_cache, сказать владельцу загрузить папки
  `app` и `tests` на GitHub. Git push самому не предлагать — владелец грузит сам.
- На каждый баг — регресс-тест; честно проверять «тест ловит баг» откатом фикса.
- Логи Railway, которые присылает владелец, — главный источник правды.
- Часовой пояс: в БД всё в UTC, дефолт владельца Europe/Moscow.
